## 5. ⚠️ Deploy awareness

`beeos-docs/.github/workflows/deploy.yml` triggers `mintlify/github-action@v4`
on every push to `main`, which **immediately publishes to docs.beeos.ai**.

**Therefore:** do **NOT** push directly to `main`. The work in this pass
is already on a feature branch `docs-roadmap-v2`. Open a PR there,
review, then merge to `main` via PR-merge (not force-push).

## 7. Rollback plan

If `mintlify validate` regresses after merging to main:

```bash
cd /Users/heat/workspace/openagent/beeos-docs
git checkout main
git revert <merge-commit-sha>
git push origin main
```

If the regression is in a specific file only, prefer a targeted fix
PR rather than a full revert.

If the issue is in the regenerated SDKs (e.g. a downstream consumer
breaks on the new types), revert the meta repo's `sdks/` directory:

